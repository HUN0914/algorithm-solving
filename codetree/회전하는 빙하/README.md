## 📝 문제 설명
2^n × 2^n 격자의 빙하를 회전시키려고 합니다.  

빙하에 속한 각각의 얼음은 인접 회전성이라는 성질을 가집니다. 좌측 상단에 있는 빙하를 특정 범위를 선택해서 회전시키게 되면 다른 상하좌우의 인접한 격자들도 똑같은 크기만큼 회전하게 됩니다.  

빙하를 회전하는 범위를 레벨이라고 표현합니다. 레벨이 L일 때 2^L × 2^L 만큼 격자를 선택하여 2^(L−1) × 2^(L−1) 만큼 잘라 4등분하여 회전시키게 됩니다.  

각각의 회전이 모두 끝나고 난 뒤에는 빙하에 속한 얼음이 녹습니다. 한 칸을 기준으로 상하좌우 인접한 칸에 얼음이 3개 이상 있는 경우에는 녹지 않습니다. 그렇지 않은 경우에는 1이 줄어듭니다.  

모든 회전을 끝내고 난 뒤에 남아있는 빙하의 총 양과 가장 큰 얼음 군집의 크기를 출력해야 합니다.  

---

## 📥 입력
- 첫째 줄에 n, q가 주어집니다.  
- 이후 2^n × 2^n 크기의 격자 정보가 주어집니다.  
- 마지막 줄에 q개의 레벨 정보가 주어집니다.  

---

## 📤 출력
- 첫째 줄에 남아있는 빙하의 총 양을 출력합니다.  
- 둘째 줄에 가장 큰 얼음 군집의 크기를 출력합니다.  

---

## 💡 접근 방법
- 레벨 L마다 2^L × 2^L 크기 블록 단위로 회전  
- 회전 후 인접 얼음 개수 기준으로 동시에 감소  
- BFS로 최대 군집 탐색

## ❌ 오답 / 실수 포인트
- **회전 직후마다 얼음을 녹여야 하는데, 이 순서를 놓치기 쉬웠다.**  
  처음에는 모든 회전을 한 번에 끝낸 뒤 마지막에만 얼음을 줄이는 방식으로 생각하기 쉬운데, 이 문제는 **레벨 하나를 회전할 때마다 바로 녹이기 연산까지 같이 수행**해야 한다.  
  즉, `회전 → 얼음 감소`를 q번 반복해야 한다.

- **얼음 감소는 동시에 처리해야 해서 원본 배열과 복사 배열을 나눠야 한다.**  
  `if (nearCnt < 3) board[i][j]--;`처럼 바로 원본 board를 줄여버리면, 뒤에 탐색하는 칸들이 이미 감소된 값을 기준으로 인접 개수를 세게 된다.  
  그래서 현재 상태는 `board`에서 읽고, 감소 결과는 `copyboard`에 반영한 뒤 마지막에 한 번에 덮어써야 한다.

- **가장 큰 얼음 군집을 구할 때 visited 초기화 위치를 잘못 잡기 쉬웠다.**  
  지금 코드처럼 시작점마다 `isVisited`를 전부 다시 false로 초기화하면, 이미 확인한 군집도 다음 시작점에서 또 다시 탐색하게 된다.  
  군집 탐색용 visited는 **전체 BFS 과정에서 공통으로 유지**되어야 하고, 한 번 방문한 얼음 칸은 다시 시작점이 되면 안 된다.  
  즉, visited는 “각 BFS마다 초기화”가 아니라 “전체 탐색 전에 한 번 초기화”가 맞다.

- **군집 탐색 시작 조건을 제대로 안 걸면 같은 덩어리를 여러 번 세게 된다.**  
  BFS를 시작할 때는  
  - 얼음이 있는 칸인지 (`board[i][j] > 0`)  
  - 아직 방문하지 않은 칸인지 (`!isVisited[i][j]`)  
  둘 다 확인해야 한다.  
  이 조건이 빠지면 이미 탐색한 군집을 다시 세거나, size가 중복 계산될 수 있다.

- **회전 구현을 좌표 공식으로 바로 처리하려다가 더 헷갈릴 수 있었다.**  
  이 문제는 부분 격자를 4등분해서  
  - 왼쪽 위 → 오른쪽 위  
  - 오른쪽 위 → 오른쪽 아래  
  - 오른쪽 아래 → 왼쪽 아래  
  - 왼쪽 아래 → 왼쪽 위  
  식으로 직접 이동시키는 방식으로 구현하면 훨씬 덜 꼬인다.  
  처음부터 전체 좌표 변환 공식을 한 번에 만들려고 하면 인덱스 실수가 나기 쉽다.

- **레벨별 회전 범위를 잡을 때, `2^L` 블록 단위로 끊어야 한다는 점이 중요했다.**  
  바깥 반복문을 `i += rotateSpace`, `j += rotateSpace`로 움직여야 하는데, 이 단위를 잘못 잡으면 블록 경계가 무너져서 회전이 아예 엉뚱하게 된다.  
  즉, “전체 격자를 한 번 회전”하는 게 아니라 **`2^L × 2^L` 부분 격자마다 따로 회전**해야 한다.

- **레벨 0 처리도 따로 신경 써야 했다.**  
  레벨 0이면 실제 회전은 일어나지 않지만, 그렇다고 전체 연산이 없는 것은 아니다.  
  **회전은 그대로 두고, 그 다음 얼음 감소는 정상적으로 수행**되어야 한다.  
  레벨 0을 통째로 건너뛰면 오답이 된다.

- **인접 칸 판정에서 격자 밖 / 얼음 0인 칸은 모두 “얼음 없음”으로 처리해야 한다.**  
  단순히 상하좌우 4칸을 다 세는 게 아니라,  
  - 범위를 벗어나면 제외  
  - 값이 0이면 제외  
  해야 한다.  
  이 조건이 조금만 흐려져도 nearCnt가 틀어져서 전체 결과가 달라진다.

- **총 얼음 양과 최대 군집 크기를 구하는 순서도 중요했다.**  
  모든 회전과 감소가 끝난 **최종 board 기준으로**  
  - 전체 합(sum)  
  - 최대 군집(BFS)  
  을 구해야 한다.  
  중간 상태에서 계산하거나, 군집 탐색 중 visited 처리 오류가 있으면 정답과 달라진다.

- **이 문제는 “구현이 맞는 것 같아 보여도 작은 순서 차이로 틀리기 쉬운 문제”였다.**  
  실제로 질문했던 부분도 대부분  
  - 회전 후 언제 감소하는지  
  - 감소를 원본에 바로 반영해도 되는지  
  - visited를 언제 초기화해야 하는지  
  - 부분 격자를 어떤 기준으로 회전해야 하는지  
  같은 구현 순서에서 나왔다.  
  그래서 이 문제는 아이디어보다도 **연산 순서를 정확히 고정하는 것**이 핵심이었다. 틀림  
- 회전 구현에서 좌표 실수 발생
